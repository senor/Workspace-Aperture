# Public Comms Playbook

Aperture's public voice should be calm, precise, and useful. The project exists to reduce workspace ambiguity, not to amplify anxiety.

## One-Liner

Aperture is a local-first workspace map for solo and agentic developers.

## Short Description

Aperture scans your local project folders, gives you Projects, Workspace, and Brief lenses, highlights reference differences and hygiene gaps, and generates compact context briefs for AI coding agents.

## Longer Description

Aperture helps solo builders manage many active, sleeping, experimental, and production projects from one local dashboard. It detects stacks, package managers, scripts, docs, Git state, env signals, CI presence, AI-readiness, and launch hygiene risks without uploading workspace data by default. The goal is not another abstract health score. The goal is a factual map you can trust before sharing a project, reviving old work, or handing context to an agent.

## Positioning Pillars

- **Local-first:** workspace facts stay on the user's machine by default.
- **Factual:** findings come from files, Git output, and scanner evidence.
- **Agent-aware:** briefs prepare Claude, Cursor, Codex, and MCP clients with concise context.
- **Read-only first:** observe and explain before mutating anything.
- **Calm operations:** surface the next useful thing without becoming a noisy alert feed.

## Issue Triage Labels

- `scanner`: factual data collection and contract shape.
- `dashboard`: UI, states, views, and local app behavior.
- `docs`: public explanation, setup, and architecture.
- `hygiene-signals`: local hygiene findings and skipped checks.
- `agent-brief`: exported context for coding agents.
- `mcp`: future read-only agent bridge.
